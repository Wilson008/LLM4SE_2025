### Differences between Grammar 1 and Grammar 2 (relevant to Instance 1)

1. **Encompass/box handling changed**
   - **Grammar 1:** `SQ_EncompassBegin` (`box ...`) and `SQ_EncompassEnd` (`end box`) were separate sequence elements.
   - **Grammar 2:** They are merged into a single block element `SQ_Encompass`:
     - `box "..." color=SQ_Color`
     - then inner elements
     - then `end box`
   - Instance text stays the same syntactically, but it now parses as one construct.

2. **Color model changed**
   - **Grammar 1:** single terminal `COLOR` (hex like `#aF7` or `#8877cc`).
   - **Grammar 2:** `SQ_Color` is either:
     - `HEXCOLOR` (same hex format), or
     - `NAMEDCOLOR` (like `#red` etc.).
   - Your instance only uses hex colors, so no instance change needed.

3. **Divider and delay rules changed**
   - **Grammar 1 Divider:** `'==' label '=='`
   - **Grammar 2 Divider:** `'==' label` (no trailing `==`)
   - **Grammar 1 Delay:** `'...' ... description ... '...'?` (optional ending `...`)
   - **Grammar 2 Delay:** `'...' ... description` (no trailing `...`)
   - **Impact:** Instance 1 must remove trailing `==` on divider lines, and remove trailing `...` on the delay line.

4. **Parameter map syntax changed (Plantestic modifications)**
   - **Grammar 1:** `PT_ParameterMap` used square brackets: `[ a: "b" ]`
   - **Grammar 2:** `PT_ParameterMap` uses curly braces: `{ a: "b" }`
   - Also, `PT_Parameter` now allows optional `: value` and values can be `ID | STRING | INT` (but we won’t instantiate new optionality per your rule).
   - **Impact:** Update all request/response parameter maps from `[...]` to `{...}` without changing spacing/formatting otherwise.

5. **Skinparam changed**
   - **Grammar 1:** `skinparam name=ID (colorval=COLOR | numberval=INT | idval=ID | strval=STRING)`
   - **Grammar 2:** `skinparam name=ID value=SQ_SkinparamValue` (same effect)
   - Instance unchanged.

6. **SQ_Description token set expanded**
   - Grammar 2 adds tokens like `'...'`, `'=='`, `':'` into description. No instance change needed.
