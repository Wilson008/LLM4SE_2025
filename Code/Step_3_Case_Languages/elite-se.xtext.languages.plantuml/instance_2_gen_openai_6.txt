## Differences between Grammar 1 and Grammar 2 (that affect the instance)

1. **Encompass/Box handling changed**
   - **Grammar 1:** `box` was split into `SQ_EncompassBegin` (`box ...`) and `SQ_EncompassEnd` (`end box`) as separate sequence elements.
   - **Grammar 2:** Replaced with a single block element `SQ_Encompass`:
     ```
     box "title" color
       (elements...)
     end box
     ```
   - Your instance already uses a `box ... end box` block form, so it still fits—no change needed.

2. **Divider syntax changed**
   - **Grammar 1:** `SQ_Divider: '==' label=SQ_Description '=='`
   - **Grammar 2:** `SQ_Divider: '==' label=SQ_Description`
   - Therefore lines like `== Initialization ==` must become `== Initialization` (remove trailing `==`).

3. **Delay syntax changed**
   - **Grammar 1:** `... (wait)? description ...?` (optional trailing `...`)
   - **Grammar 2:** `... (wait)? description` (no trailing `...`)
   - Therefore `... wait(10 s) Auf schönes Wetter ;) ...` must become `... wait(10 s) Auf schönes Wetter ;)` (remove trailing `...`).

4. **Parameter map syntax changed**
   - **Grammar 1:** `PT_ParameterMap` used **square brackets**: `[ ... ]`
   - **Grammar 2:** `PT_ParameterMap` uses **curly braces**: `{ ... }`
   - Also:
     - **Grammar 1:** parameter required `name: "value"`
     - **Grammar 2:** parameter is `name (':' value)?` and value may be `ID | STRING | INT`
   - So you must replace request/response parameter maps from `[...]` to `{...}`. Keep the same parameters (don’t add optional ones).

5. **Colors refactored**
   - Grammar 2 introduces `SQ_Color` = `HEXCOLOR | NAMEDCOLOR` and replaces `COLOR` usages.
   - Your instance uses hex colors like `#8877cc`, `#eba`, `#aF7`, `#0000aa`, etc., which still match `HEXCOLOR`.
   - No instance change required.

6. **Skinparam rule slightly refactored**
   - **Grammar 1:** `skinparam name=ID (colorval=COLOR | numberval=INT | idval=ID | strval=STRING)`
   - **Grammar 2:** `skinparam name=ID value=SQ_SkinparamValue` where value can be color/int/id/string.
   - Your line `skinparam roundcorner 25` still fits—no change needed.

7. **SQ_Description token set extended**
   - Grammar 2 adds tokens like `'...'`, `'=='`, `':'` to `SQ_Description`.
   - This is permissive; no changes required.
