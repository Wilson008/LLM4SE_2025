## Differences: Grammar 2 vs Grammar 1

### 1) Top-level elements inside `Contract`
- **Grammar 1** allowed:
  - `attributes+=Attribute | events+=Event | methods+=Method | modifiers+=Modifier`
- **Grammar 2** allows:
  - `attributes+=Attribute | events+=Event | methods+=Method | validators+=Validator | dataStores+=DataStore`

So:
- `Modifier` declarations were **removed** from the contract body and replaced by **Validator** declarations.
- A new top-level construct **DataStore** was added.

### 2) Renaming/reframing: `Modifier` → `Validator`
- Rule name changed from **`Modifier`** to **`Validator`**
- Keyword changed from **`'Modifier'`** to **`'validator'`** (note the lowercase keyword in Grammar 2)
- Internal structure is otherwise the same:
  - name, optional params, and body with `message:` and `validation:`.

This is both a **syntactic change** (different keyword) and a **metamodel change** (different EClass/type name).

### 3) Method references: `modifiers:` → `validators:`
In `Method`:
- Grammar 1:
  - `modifiersKeyword='modifiers:' modifiers+=[Modifier] (',' modifiers+=[Modifier])*` (optional)
- Grammar 2:
  - `modifiersKeyword='validators:' validators+=[Validator] (',' validators+=[Validator])*` (optional)

So any instance using `modifiers:` and referencing `Modifier` names must be updated to `validators:` and reference `validator` definitions.

### 4) New construct: `DataStore`
Grammar 2 adds:
```xtext
DataStore:
  'dataStore' name=ID 'fromType' fromType=DataType 'toType' toType=DataType
;
```
This introduces a new kind of declaration in `Contract` that maps one `DataType` to another.

### 5) Everything else is effectively unchanged
- `Contract` header fields (`version`, `hasReceive`, `ownership`) remain.
- `Attribute`, `Event`, `Param`, `Output`, and `DataType` enum stay the same (same literals, same structure).
- `Method` signature format is the same (params, optional `stateMutability:`, optional outputs list, optional description/events).

### 6) Compatibility impact on Instance 1
Given Instance 1 (Grammar 1):
- `Modifier onlyOwner ...` and `Modifier validNumber ...` would no longer parse under Grammar 2:
  - must become `validator onlyOwner ...` etc.
- Inside `method example { ... }`
  - `modifiers: onlyOwner, validNumber` must become `validators: onlyOwner, validNumber`
- If you add `dataStore ...` lines, those would only be valid in Grammar 2.
