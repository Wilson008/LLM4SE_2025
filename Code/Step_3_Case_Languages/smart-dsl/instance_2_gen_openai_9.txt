Differences between Grammar 1 and Grammar 2 (that affect instances)

1. **`Modifier` rule renamed to `Validator`**
   - Grammar 1: `Modifier: 'Modifier' name=ID ...`
   - Grammar 2: `Validator: 'validator' name=ID ...`
   - This changes both the keyword (`'Modifier'` â†’ `'validator'`) and the referenced type name in methods.

2. **In `Contract`, the repeated elements changed**
   - Grammar 1 allowed: `attributes | events | methods | modifiers`
   - Grammar 2 allows: `attributes | events | methods | validators | dataStores`
   - `dataStores` is new and optional (in an `|`), so per your rule (2) it should **not** be instantiated.

3. **In `Method`, the keyword for linking to modifiers changed**
   - Grammar 1: `modifiersKeyword='modifiers:' modifiers+=[Modifier] ...`
   - Grammar 2: `modifiersKeyword='validators:' validators+=[Validator] ...`
   - So inside method bodies: `modifiers:` must become `validators:` and the referenced names must refer to `validator` declarations.

4. **Everything else used by Instance 1 remains compatible**
   - `Contract`, `Attribute`, `Event`, `Param`, `Output`, and `DataType` are unchanged in the parts exercised by Instance 1.
   - `DataStore` is new but optional, so we do not add it.

## Instance 2 (updated to conform to Grammar 2)

Contract SimpleContract {
	version: "v1.0.0"
	hasReceive
	ownership
	modifiable total integer
	
	validator onlyOwner () {
		message: "Only the contract owner can call this function."
		validation: "msg.sender == owner"
	}
	
	validator validNumber(x integer) {
		message: "The number must be greater than 4"
		validation: "x>4"
	}
	
	method example (x integer, name string, lastname string) -> x integer, name string {
		description: "This is a sample of method"
		validators: onlyOwner, validNumber
		events: XUpdated, NameUpdated 
	}
	
	event XUpdated (x integer){
		description: "Este evento es para notificar cuando el numero se ha actualizado"
	}
	
	event NameUpdated(name string){
		description: "Este evento es para notificar cuando el name se ha actualizado"
	}
}
