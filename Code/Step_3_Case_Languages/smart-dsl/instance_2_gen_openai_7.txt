Differences between Grammar 1 and Grammar 2 (relevant to evolving the instance)

1. **`Modifier` renamed to `Validator`**
   - Grammar 1: `Modifier: 'Modifier' ...`
   - Grammar 2: `Validator: 'validator' ...`
   - This affects:
     - The keyword introducing the block (`'Modifier'` → `'validator'`)
     - The collection stored in `Contract` (`modifiers+=Modifier` → `validators+=Validator`)

2. **Method section label changed**
   - Grammar 1 method body: `modifiersKeyword='modifiers:' modifiers+=[Modifier] ...`
   - Grammar 2 method body: `modifiersKeyword='validators:' validators+=[Validator] ...`
   - So inside `method { ... }`, the label **must** be `validators:` instead of `modifiers:`.
   - References must point to `Validator` names (same names still work, just different rule type).

3. **New optional/OR elements added (not instantiated per your rule #2)**
   - Grammar 2 adds `dataStores+=DataStore` in the Contract `( ... | dataStores+=DataStore)*`
   - New rule `DataStore: 'dataStore' ...`
   - Since it’s new and appears via `|` in the contract body, **we do not add any `dataStore` lines**.

4. Everything else used in Instance 1 remains compatible (Contract header, `version:`, `hasReceive`, `ownership`, `Attribute`, `Event`, `Method`, params/outputs, datatype enum unchanged).

### Instance 2 (conforms to Grammar 2)

```txt
Contract SimpleContract {
	version: "v1.0.0"
	hasReceive
	ownership
	modifiable total integer
	
	validator onlyOwner () {
		message: "Only the contract owner can call this function."
		validation: "msg.sender == owner"
	}
	
	validator validNumber(x integer) {
		message: "The number must be greater than 4"
		validation: "x>4"
	}
	
	method example (x integer, name string, lastname string) -> x integer, name string {
		description: "This is a sample of method"
		validators: onlyOwner, validNumber
		events: XUpdated, NameUpdated 
	}
	
	event XUpdated (x integer){
		description: "Este evento es para notificar cuando el numero se ha actualizado"
	}
	
	event NameUpdated(name string){
		description: "Este evento es para notificar cuando el name se ha actualizado"
	}
}
```
