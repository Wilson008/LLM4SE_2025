## Differences: Grammar 2 vs Grammar 1

### 1) Contract contents (top-level elements)
- **Grammar 1** allows, inside `Contract`:
  - `attributes+=Attribute | events+=Event | methods+=Method | modifiers+=Modifier`
- **Grammar 2** changes this to:
  - `attributes+=Attribute | events+=Event | methods+=Method | validators+=Validator | dataStores+=DataStore`

So:
- `Modifier` is removed from the contract’s top-level alternatives.
- `Validator` and `DataStore` are added.

### 2) Modifier renamed/replaced by Validator
- **Grammar 1 rule:** `Modifier: 'Modifier' name=ID ...`
  - Keyword is capitalized: **`Modifier`**
- **Grammar 2 rule:** `Validator: 'validator' name=ID ...`
  - Keyword is lowercase: **`validator`**
- Internals are structurally the same:
  - params list
  - block with `message:` and `validation:`

### 3) Methods now reference validators (and keyword changes)
- **Grammar 1 Method**:
  - uses `modifiersKeyword='modifiers:' modifiers+=[Modifier] ...`
- **Grammar 2 Method**:
  - uses `modifiersKeyword='validators:' validators+=[Validator] ...`

So both the label in the method body and the referenced type change:
- `modifiers:` → `validators:`
- `[Modifier]` → `[Validator]`

(Variable name `modifiersKeyword` remains, but it now captures the literal `validators:`—a naming inconsistency, but it will still work.)

### 4) New rule: DataStore
- Grammar 2 adds:
  ```xtext
  DataStore:
    'dataStore' name=ID 'fromType' fromType=DataType 'toType' toType=DataType
  ;
  ```
This introduces a new top-level construct in `Contract`, describing a mapping/relationship from one `DataType` to another.

### 5) Everything else is unchanged
- `Attribute`, `Event`, `Param`, `Output`, and `DataType` enum remain effectively the same.
- `Contract` header fields (`version`, optional `hasReceive`, optional `ownership`) remain the same.
- `Method` signature structure remains the same (params, optional `stateMutability:`, outputs, optional description, events list).

## Impact on Instance 1 (Grammar 1 instance)
Instance 1 would **not** parse under Grammar 2 without edits because:
- `Modifier ...` blocks must become `validator ...` blocks (and keyword case changes).
- Inside the method body, `modifiers:` must become `validators:`.
- Any `Modifier` references must become `Validator` references (same names can be reused, but they must be declared via `validator`).
