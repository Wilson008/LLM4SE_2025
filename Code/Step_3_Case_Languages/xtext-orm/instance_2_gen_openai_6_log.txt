### Differences: Grammar 2 vs Grammar 1

#### 1) Top-level structure / model header
- **Grammar 1:** A model is just one or more entities:
  - `PersistenceModel: (entities+=Entity)+;`
- **Grammar 2:** A model **must start with a configuration block**, then entities:
  - `PersistenceModel: 'Configuration' '[' configs+=Configuration+ ']' (entities+=Entity)+;`
  - This introduces **mandatory configuration(s)** before any entities.

#### 2) Field options syntax moved to a wrapper rule
- **Grammar 1:** Options (`[unique, not_null]`) are attached **inside each field rule**:
  - `PropertyField ... ('[' options+=TypeOption ... ']')?;`
  - `EntityField ... ('[' options+=TypeOption ... ']')?;`
  - `BackrefField ... ('[' options+=TypeOption ... ']')?;`
- **Grammar 2:** Options are removed from the individual field types and handled uniformly via:
  - `FieldDeclaration: field=Field ('[' options+=TypeOption ... ']')?;`
  - Entity now contains `fieldDeclarations+=FieldDeclaration+` instead of `fields+=Field+`.
- Effect: **Any field kind** can now have options in a consistent way (as long as it’s wrapped in `FieldDeclaration`).

#### 3) Entity relationships extended with directionality
- **Grammar 1 `EntityField`:**
  - `object [cardinality] name: EntityRef ...`
- **Grammar 2 `EntityField`:**
  - `object '[' cardinality=Cardinality '|' (bidirectional ?= '<->'| unidirectional ?= '->') ']' name=ID ':' entityReference=[Entity]`
- Effect:
  - Relationship declarations now require **both**:
    - a **cardinality**, and
    - an explicit **direction** (`<->` or `->`)
  - Example new bracket form: `object [1..n|->] teams: Team`

#### 4) Backref and property/entity fields no longer carry options directly
- Because options moved to `FieldDeclaration`, the specific rules became simpler:
  - `PropertyField` now: `'property' name=ID ':' type=Type`
  - `EntityField` now: no trailing options part
  - `BackrefField` now: no trailing options part
- Practically: your instances must attach options after the `Field` via the surrounding `FieldDeclaration`.

#### 5) New configuration subsystem added
Grammar 2 adds:
- `Configuration: JavaConfiuration | PythonConfiguration;`
- Java config block with:
  - `genpath` (a `Path`)
  - `package` (an `FQN`)

- Python config block with:
  - `genpath` (a `Path`)
  - `module_name` (an `FQN`)
  - `ORM` (currently only `SQLAlchemy`)
- New `Path` rule: `ID ('/' ID)*` (slash-separated segments)
