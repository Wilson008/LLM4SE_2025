## Differences: Grammar 1 → Grammar 2

### 1) New top-level configuration section
- **Grammar 1:** `PersistenceModel` contains only entities:
  `(entities+=Entity)+`
- **Grammar 2:** Adds a required configuration block before entities:
  ```
  'Configuration' '[' configs+=Configuration+ ']' (entities+=Entity)+
  ```
  So every model must now start with `Configuration [...]`.

### 2) Options moved out of individual field rules into a wrapper
- **Grammar 1:** Each field type could carry options directly:
  - `PropertyField ... ('[' options+=TypeOption ... ']')?`
  - `EntityField ... ('[' options+=TypeOption ... ']')?`
  - `BackrefField ... ('[' options+=TypeOption ... ']')?`
- **Grammar 2:** Options are attached at the `FieldDeclaration` level:
  ```
  FieldDeclaration:
    field=Field ('[' options+=TypeOption (',' options+=TypeOption)? ']')?
  ;
  ```
  And `Entity` now contains `fieldDeclarations+=FieldDeclaration+` (instead of `fields+=Field+`).

**Effect:** Syntactically, options are still written after a field, but the AST structure changes: options belong to `FieldDeclaration`, not to the concrete field (`PropertyField`/`EntityField`/`BackrefField`).

### 3) EntityField now encodes directionality (uni-/bi-directional)
- **Grammar 1 `EntityField`:**
  ```
  'object' '[' cardinality=Cardinality ']' name=ID ':' entityReference=[Entity] ...
  ```
- **Grammar 2 `EntityField`:**
  ```
  'object' '[' cardinality=Cardinality '|' (bidirectional ?= '<->'| unidirectional ?= '->') ']' name=ID ':' entityReference=[Entity]
  ```
  **New required syntax:** `object [<cardinality>|<->] ...` or `object [<cardinality>|->] ...`.

### 4) BackrefField options handling changed (same syntax, different owner)
- **Grammar 1:** backref itself could have `[...]` options.
- **Grammar 2:** `BackrefField` has no options directly; options come via `FieldDeclaration`.

### 5) New configuration language elements
Grammar 2 introduces:
- `Configuration: JavaConfiuration | PythonConfiguration;`
- Java config block with:
  - `genpath` (a `Path`)
  - `package` (a `FQN`)
  - `ORM` (enum currently only `Hibernate`)
- Python config block with:
  - `genpath` (a `Path`)
  - `module_name` (a `FQN`)
  - `ORM` (enum currently only `SQLAlchemy`)
- New `Path` rule: `ID ('/' ID)*`

### 6) Minor structural renames / refactors
- In `Entity`: `fields` → `fieldDeclarations` containing `FieldDeclaration` wrappers.
- `Model`, `Type`, `FQN`, `Cardinality`, `TypeOption` and the concrete types (`String`, `Integer`) are otherwise essentially unchanged.
