### Differences between Grammar 1 and Grammar 2 (that affect the instance)

**Network section keyword changes (inside single quotes):**
- `'batch-size'` → `'batchSize'`
- `'image-size'` → `'imageSize'`
- `'image-channels'` → `'imageChannels'`
- `'output-labels'` → `'outputLabels'`
- `'caffe-path'` → `('caffePath' ...)?` (**now optional**)
- `'output-path'` → `'outputPath'`
- `'train-data-path'` → `'trainDataPath'` (optional in both; not used in Instance 1)
- `'val-data-path'` → `'valDataPath'` (optional in both; not used in Instance 1)

**Conv/Norm/Scale keyword changes:**
- In `ConvLayerBody`: `'kernel-size'` → `'kernelSize'`
- In `BachNormBody`: `'use-global-stats'` → `'useGlobalStats'`
- In `ScaleNormBody`: `'use-bias-term'` → `'useBiasTerm'`

**Branch body change:**
- Grammar 1 `BranchBody` had both:
  - `'type' -> BranchType` and `'operation' -> OperationType`
- Grammar 2 `BranchBody` only has:
  - `'eltwiseOperation' -> OperationType`
- Also, `BranchType` enum is removed in Grammar 2.

**LayerName literal change:**
- `'final-layer'` → `'finalLayer'`

**Important constraint you gave:** since `caffePath` is **optional** in Grammar 2 and was mandatory in Grammar 1, I must **not** instantiate it in Instance 2.
