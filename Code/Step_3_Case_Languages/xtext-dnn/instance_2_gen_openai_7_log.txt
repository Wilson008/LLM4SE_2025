## Differences: Grammar 2 vs Grammar 1

### 1) Network keyword/property renames (kebab-case → camelCase)
In `Network` rule, these literals changed:

- `image-size` → `imageSize`
- `image-channels` → `imageChannels`
- `batch-size` → `batchSize`
- `output-labels` → `outputLabels`
- `train-data-path` → `trainDataPath`
- `val-data-path` → `valDataPath`
- `caffe-path` → `caffePath`
- `output-path` → `outputPath`

`epochs`, `seed`, `updater`, `learningRate` stayed the same literal spellings as before (except they’re already camelCase / same).

### 2) `caffePath` became optional
- Grammar 1: `'caffe-path' ASSIGN caffePath=STRING &` **required**
- Grammar 2: `('caffePath' ASSIGN caffePath=STRING)? &` **optional**

Everything else in `Network` remains required/optional as previously, except for the renamed keywords above.

### 3) Layer/body keyword renames (kebab-case → camelCase)
- `ConvLayerBody`: `kernel-size` → `kernelSize`
- `BachNormBody`: `use-global-stats` → `useGlobalStats`
- `ScaleNormBody`: `use-bias-term` → `useBiasTerm`

### 4) Branch syntax changed (removed BranchType/type field; renamed operation field)
**Grammar 1 `BranchBody`:**
- required: `'type' -> BranchType`
- required: `'operation' -> OperationType`

Also had `enum BranchType` with `RESIDUAL='residual'`.

**Grammar 2 `BranchBody`:**
- only: `'eltwiseOperation' -> OperationType`
- **BranchType enum removed entirely**
- No `type -> residual` (or any other branch type) in the grammar anymore.

So branch blocks are simpler now: they only specify the eltwise operation.

### 5) LayerName special token renamed
- `final-layer` → `finalLayer`

This affects `LayerName`:
- Grammar 1: `name=(STRING | 'final-layer')`
- Grammar 2: `name=(STRING | 'finalLayer')`

### 6) What did *not* change
- `Layer` alternatives and overall structure are essentially the same (conv/pool/dense/norm/scale/branch).
- `PoolLayerBody` still uses `'type' -> PoolingType` and optional `'stride' = INTEGER`.
- Enums largely unchanged (UpdaterType, ActivationFunctionType, WeightInitType, PoolingType, OperationType, Decission).
- Terminals (INTEGER/REAL/etc.) unchanged.
