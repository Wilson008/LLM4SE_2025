### Differences: Grammar 2 vs Grammar 1

#### 1) Network keyword/property names changed (kebab-case → camelCase)
In `Network`:

- `image-size` → `imageSize`
- `image-channels` → `imageChannels`
- `batch-size` → `batchSize`
- `output-labels` → `outputLabels`
- `train-data-path` → `trainDataPath`
- `val-data-path` → `valDataPath`
- `caffe-path` → `caffePath`
- `output-path` → `outputPath`

Implication: all instances must use the new spellings to parse under Grammar 2.

#### 2) `caffePath` is no longer required
- **Grammar 1:** `'caffe-path' ASSIGN caffePath=STRING` (mandatory)
- **Grammar 2:** `('caffePath' ASSIGN caffePath=STRING)?` (optional)

So an instance can omit `caffePath` in Grammar 2.

#### 3) Conv layer body keys changed
- `kernel-size` → `kernelSize`
- `stride` remains `stride`

#### 4) Batch norm / scale keys changed
- `use-global-stats` → `useGlobalStats`
- `use-bias-term` → `useBiasTerm`

#### 5) Branch body redesigned
- **Grammar 1 BranchBody:**
  - `'type' -> BranchType` **and**
  - `'operation' -> OperationType`
  - plus enums: `BranchType` existed (`residual`)
- **Grammar 2 BranchBody:**
  - only `'eltwiseOperation' -> OperationType`
  - `BranchType` enum is removed entirely

Implications for instances:
- You must replace:
  - `operation -> PROD` with `eltwiseOperation -> PROD`
- You must remove:
  - `type -> residual` (no longer valid)
- More generally: Grammar 2 no longer models branch “type”, only the eltwise operation.

#### 6) “final layer” token changed
- In `LayerName`:
  - Grammar 1 allows `'final-layer'`
  - Grammar 2 allows `'finalLayer'`

So `name: final-layer` must become `name: finalLayer` under Grammar 2.

#### 7) Everything else is effectively unchanged
Layer alternatives, pooling syntax, dense/conv/norm/scale constructs, enums (except `BranchType` removal), and terminals are otherwise the same in structure and spelling.
