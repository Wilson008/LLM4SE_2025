## Differences: Grammar 2 vs Grammar 1

### 1) Network property keywords renamed (kebab-case → camelCase)
In `Network:` these literals changed:

- `image-size` → `imageSize`
- `image-channels` → `imageChannels`
- `batch-size` → `batchSize`
- `output-labels` → `outputLabels`
- `train-data-path` → `trainDataPath`
- `val-data-path` → `valDataPath`
- `caffe-path` → `caffePath`
- `output-path` → `outputPath`

Also: `epochs`, `updater`, `learningRate`, `seed` stayed the same.

### 2) `caffePath` is no longer required
- Grammar 1: `'caffe-path' ASSIGN caffePath=STRING` (mandatory)
- Grammar 2: `('caffePath' ASSIGN caffePath=STRING)?` (optional)

Everything else in the `Network` header remains required as before (except the already-optional train/val paths and seed).

### 3) Branch body structure changed (and BranchType removed)
Grammar 1 `BranchBody`:
- had **two** entries:
  - `'type' -> BranchType`
  - `'operation' -> OperationType`
- and `enum BranchType { RESIDUAL='residual' }`

Grammar 2 `BranchBody`:
- has **only one** entry:
  - `'eltwiseOperation' -> OperationType`
- `BranchType` enum is entirely removed, and the `type -> residual` concept disappears.

So branch blocks must now specify `eltwiseOperation -> PROD|SUM` and can’t specify `type -> residual`.

### 4) Layer/body keywords renamed to camelCase
- `ScaleNormBody`: `use-bias-term` → `useBiasTerm`
- `BachNormBody`: `use-global-stats` → `useGlobalStats`
- `ConvLayerBody`: `kernel-size` → `kernelSize`
- `LayerName`: `final-layer` → `finalLayer`

Other layer/body keywords (e.g., `activation`, `weightsInit`, `biasInit`, `dropout`, pool `type`, `stride`) remain unchanged.

### 5) Unchanged parts (important for compatibility expectations)
- Layer alternatives (`conv`, `pool`, `dense`, `norm`, `scale`, `branch`) are structurally the same except for the branch body field changes and renamed literals above.
- Enums still present and unchanged: `Decission`, `OperationType`, `PoolingType`, `ActivationFunctionType`, `UpdaterType`, `WeightInitType` (except `BranchType` removed).
- Terminals (`INTEGER`, `REAL`, `ASSIGN`, `REFERENCE`, etc.) unchanged.
