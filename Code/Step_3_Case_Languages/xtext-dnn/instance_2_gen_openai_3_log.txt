## Differences between Grammar 1 and Grammar 2

### 1) Network header keywords renamed (kebab-case → camelCase)
In `Network`:

- `image-size` → `imageSize`
- `image-channels` → `imageChannels`
- `batch-size` → `batchSize`
- `output-labels` → `outputLabels`
- `train-data-path` → `trainDataPath`
- `val-data-path` → `valDataPath`
- `caffe-path` → `caffePath`
- `output-path` → `outputPath`

Effect: Instance files written for Grammar 1 using the old dashed keywords won’t parse under Grammar 2 unless updated.

### 2) `caffePath` changed from required to optional
- Grammar 1: `'caffe-path' ASSIGN caffePath=STRING &` **required**
- Grammar 2: `('caffePath' ASSIGN caffePath=STRING)? &` **optional**

Effect: Grammar 2 allows networks without specifying `caffePath`.

### 3) Branch body structure changed
**Grammar 1 `BranchBody`:**
- had **two** fields:
  - `'type' -> BranchType`
  - `'operation' -> OperationType`
- plus an enum `BranchType` with `RESIDUAL='residual'`

**Grammar 2 `BranchBody`:**
- now only:
  - `'eltwiseOperation' -> OperationType`
- and `BranchType` enum is **removed**

Effect: In Grammar 2, branch blocks no longer accept/require `type -> residual`, and `operation -> PROD` must be replaced with `eltwiseOperation -> PROD`.

### 4) Layer/parameter keywords renamed
- In `ScaleNormBody`:
  - `use-bias-term` → `useBiasTerm`
- In `BachNormBody`:
  - `use-global-stats` → `useGlobalStats`
- In `ConvLayerBody`:
  - `kernel-size` → `kernelSize`

Effect: Corresponding layer bodies from Grammar 1 instances must be updated.

### 5) Special layer name literal renamed
In `LayerName`:
- Grammar 1 allows `'final-layer'`
- Grammar 2 allows `'finalLayer'`

Effect: `name: final-layer` in instances must become `name: finalLayer` to parse under Grammar 2.

### 6) Everything else is effectively unchanged
- `Layer` alternatives and overall structure are the same (conv/pool/dense/norm/scale/branch).
- `PoolLayerBody` still uses `type -> MAX|AVG` and optional `stride`.
- Enums for `Decission`, `OperationType`, `PoolingType`, `ActivationFunctionType`, `UpdaterType`, `WeightInitType` are the same (except removal of `BranchType`).
- Terminals (`INTEGER`, `REAL`, `ASSIGN`, `REFERENCE`, etc.) unchanged.
