Differences between Grammar 1 and Grammar 2 (relevant to the instance):

1. **`MongoBean` syntax changed**
   - **Grammar 1:** `MongoBean: 'mongobean' name=ValidID '{' ... '}' ;`
   - **Grammar 2:** `MongoBean: name=ValidID '{' ... '}' ;`
   - So the literal keyword **`'mongobean'` is removed** from every bean declaration.

2. **`MongoProperty` was extended with an optional/OR alternative**
   - **Grammar 1:** `type=JvmTypeReference (many?='*')? name=ValidID;`
   - **Grammar 2:** `(type=JvmTypeReference | inlineType=MongoBean) (many?='*')? name=ValidID;`
   - This adds an **OR** option (`inlineType=MongoBean`), which is new and should **not** be instantiated per your rule (2). So all properties remain using normal `JvmTypeReference` as before.

3. Everything else used by the instance (imports, package, operations, blocks, etc.) remains syntactically the same for this text.

			seconds = Integer::parseInt(split.get(0)) * 60 + Integer::parseInt(split.get(1))
		}		
		
		String toString() {
			Integer::toString(seconds / 60) + ':' + Integer::toString(seconds % 60)
		}

			new Duration() [

			]
		}
	}
