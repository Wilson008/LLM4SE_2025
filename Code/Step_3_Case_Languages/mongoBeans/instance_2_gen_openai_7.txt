Differences between **Grammar 1** and **Grammar 2** (that affect the concrete syntax):

1. **`MongoBean` header changed**
   - **Grammar 1:** `MongoBean: 'mongobean' name=ValidID '{' ... '}' ;`
   - **Grammar 2:** `MongoBean: name=ValidID '{' ... '}' ;`
   - Impact: the keyword **`'mongobean'` is removed** from bean declarations. So every occurrence of `mongobean X {` must become `X {`.

2. **`MongoProperty` type was extended (optional/OR)**
   - **Grammar 1:** `type=JvmTypeReference (many?='*')? name=ValidID;`
   - **Grammar 2:** `(type=JvmTypeReference | inlineType=MongoBean) (many?='*')? name=ValidID;`
   - Impact: properties can now use an inline bean type as an alternative, but since this is an **OR extension**, per your rule (2) we **do not instantiate** the new alternative. Existing properties using `JvmTypeReference` remain unchanged.

3. Everything else relevant to Instance 1 stays the same (imports, package declaration, operations syntax, `*`, parentheses/braces, comments/formatting).

import java.util.*

package org.musicdb {
	/**
	 * A musical performer
	 */
	Artist {
		String name
		Album* albums
		
		Iterable<Track> getOevre() {
			albums.map[tracks].flatten
		}
	}  
	 
	/**
	 * A record, CD, DVD or another collection of tracks
	 */
	Album {
		String title
		int year 
		Track* tracks
		
		Duration getDuration() {
			tracks.map[duration].reduce[Duration a, Duration b| a + b]
		}
	}
	
	Track {
		String title
		Duration duration
	}
	
	Duration {
		int seconds
	
		void fromString(String duration) {
			val split = duration.split(':')
			seconds = Integer::parseInt(split.get(0)) * 60 + Integer::parseInt(split.get(1))
		}		
		
		String toString() {
			Integer::toString(seconds / 60) + ':' + Integer::toString(seconds % 60)
		}
		
		Duration operator_plus(Duration other) {
			new Duration() [
				seconds = this.seconds + other.seconds
			]
		}
	}
}
