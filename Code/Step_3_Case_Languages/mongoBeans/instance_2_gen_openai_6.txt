### Differences between Grammar 1 and Grammar 2

1. **`MongoBean` rule changed**
   - **Grammar 1:** `MongoBean` instances must start with the keyword **`'mongobean'`**:
     ```
     'mongobean' name=ValidID '{' ... '}'
     ```
   - **Grammar 2:** the **`'mongobean'`** keyword was removed; now a bean starts directly with the `name`:
     ```
     name=ValidID '{' ... '}'
     ```

2. **`MongoProperty` rule changed**
   - **Grammar 1:** a property must have a `type=JvmTypeReference`.
   - **Grammar 2:** a property can have either a normal JVM type **or** an inline bean type:
     ```
     (type=JvmTypeReference | inlineType=MongoBean) (many?='*')? name=ValidID;
     ```
   - Per your rule (2), since this is an **OR** addition, we **do not** instantiate the new option (`inlineType=MongoBean`). So we keep all properties using the existing `type=JvmTypeReference` form.

3. Everything else remains the same (imports, packages, operations, formatting constructs).

---

### Instance 2 (updated to follow Grammar 2)

Only the **`mongobean`** keyword is removed everywhere it appears; all spacing, indentation, comments, and line breaks are preserved exactly otherwise.

```
// Instance 2
import java.util.*

package org.musicdb {
	/**
	 * A musical performer
	 */
	Artist {
		String name
		Album* albums
		
		Iterable<Track> getOevre() {
			albums.map[tracks].flatten
		}
	}  
	 
	/**
	 * A record, CD, DVD or another collection of tracks
	 */
	Album {
		String title
		int year 
		Track* tracks
		
		Duration getDuration() {
			tracks.map[duration].reduce[Duration a, Duration b| a + b]
		}
	}
	
	Track {
		String title
		Duration duration
	}
	
	Duration {
		int seconds
	
		void fromString(String duration) {
			val split = duration.split(':')
			seconds = Integer::parseInt(split.get(0)) * 60 + Integer::parseInt(split.get(1))
		}		
		
		String toString() {
			Integer::toString(seconds / 60) + ':' + Integer::toString(seconds % 60)
		}
		
		Duration operator_plus(Duration other) {
			new Duration() [
				seconds = this.seconds + other.seconds
			]
		}
	}
} 
```
