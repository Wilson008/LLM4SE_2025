Differences between Grammar 1 and Grammar 2
- **MongoBean header changed**:
  - Grammar 1: `'mongobean' name=ValidID '{' ... '}'`
  - Grammar 2: `name=ValidID '{' ... '}'`
  - Meaning: the literal keyword **`'mongobean'` is removed**, so bean declarations now start directly with the bean name.
- **MongoProperty type changed (extended)**:
  - Grammar 1: `type=JvmTypeReference (many?='*')? name=ValidID;`
  - Grammar 2: `(type=JvmTypeReference | inlineType=MongoBean) (many?='*')? name=ValidID;`
  - Meaning: properties can now either reference a JVM type **or** contain an inline `MongoBean`. Since this is an **OR** addition, per your rule (2) we **do not** introduce inline beans in the evolved instance.
