Differences between Grammar 1 and Grammar 2 that affect instances:

1. **`MongoBean` header changed**
   - **Grammar 1:** `mongobean` keyword is required:  
     `MongoBean: 'mongobean' name=ValidID '{' ... '}' ;`
   - **Grammar 2:** the `'mongobean'` keyword was removed:  
     `MongoBean: name=ValidID '{' ... '}' ;`
   - Impact: in the instance, every `mongobean X {` must become `X {`.

2. **`MongoProperty` type extended (OR alternative added)**
   - **Grammar 1:** only `type=JvmTypeReference`
   - **Grammar 2:** `(type=JvmTypeReference | inlineType=MongoBean)` allowing inline bean types.
   - Per your rule (2), since this is a new optional/OR capability, we **do not instantiate** the new `inlineType=MongoBean` form. So properties stay as they were (using normal `JvmTypeReference` types like `String`, `Album`, etc.).

Everything else remains the same for this instance (imports, package, operations, bodies, comments, whitespace).

### Instance 2 (conforms to Grammar 2)

```
// Instance 2
import java.util.*

package org.musicdb {
	/**
	 * A musical performer
	 */
	Artist {
		String name
		Album* albums
		
		Iterable<Track> getOevre() {
			albums.map[tracks].flatten
		}
	}  
	 
	/**
	 * A record, CD, DVD or another collection of tracks
	 */
	Album {
		String title
		int year 
		Track* tracks
		
		Duration getDuration() {
			tracks.map[duration].reduce[Duration a, Duration b| a + b]
		}
	}
	
	Track {
		String title
		Duration duration
	}
	
	Duration {
		int seconds
	
		void fromString(String duration) {
			val split = duration.split(':')
			seconds = Integer::parseInt(split.get(0)) * 60 + Integer::parseInt(split.get(1))
		}		
		
		String toString() {
			Integer::toString(seconds / 60) + ':' + Integer::toString(seconds % 60)
		}
		
		Duration operator_plus(Duration other) {
			new Duration() [
				seconds = this.seconds + other.seconds
			]
		}
	}
}
```
