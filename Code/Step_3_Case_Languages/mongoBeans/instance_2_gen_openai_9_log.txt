## Key differences: Grammar 2 vs Grammar 1

### 1) `mongobean` keyword removed from `MongoBean`
- **Grammar 1**
  ```xtext
  MongoBean:
    'mongobean' name=ValidID '{' ... '}';
  ```
- **Grammar 2**
  ```xtext
  MongoBean:
    name=ValidID '{' ... '}';
  ```
**Effect:** In Grammar 2, a bean is declared just by an identifier followed by `{ ... }` (e.g., `Artist { ... }`) instead of `mongobean Artist { ... }`.

---

### 2) Properties can now use an *inline/nested* `MongoBean` type
- **Grammar 1**
  ```xtext
  MongoProperty:
    type=JvmTypeReference (many?='*')? name=ValidID;
  ```
- **Grammar 2**
  ```xtext
  MongoProperty:
    (type=JvmTypeReference | inlineType=MongoBean) (many?='*')? name=ValidID;
  ```
**Effect:** A property type can be either:
- a normal JVM type reference (`String`, `int`, `Iterable<Track>`, etc.), **or**
- an inline bean definition used as a type, e.g. conceptually:
  ```
  Address { String street } address
  ```
This introduces nested/anonymous-ish bean types directly inside property declarations.

---

### 3) Everything else is effectively unchanged
- `MongoFile`, `Import`, `PackageDeclaration`, `AbstractFeature`, `MongoOperation`, and `QualifiedNameWithWildcard` are structurally the same.
- `MongoOperation` signature/guard (`=>`) and body (`XBlockExpression`) are unchanged.
