Differences between **Grammar 1** and **Grammar 2** that affect the concrete syntax:

1. **MongoBean rule changed**
   - **Grammar 1:** `MongoBean: 'mongobean' name=ValidID '{' ... '}';`
   - **Grammar 2:** `MongoBean: name=ValidID '{' ... '}';`
   - Effect: the literal keyword **`'mongobean'` is removed**, so each bean declaration must start directly with the name.

2. **MongoProperty rule extended (optional/OR)**
   - **Grammar 1:** `type=JvmTypeReference (many?='*')? name=ValidID;`
   - **Grammar 2:** `(type=JvmTypeReference | inlineType=MongoBean) (many?='*')? name=ValidID;`
   - Effect: properties may alternatively use an inline bean type, but since this is an **OR addition**, per your rule (2) we **do not instantiate** the new alternative. So we keep using `JvmTypeReference` as before (e.g., `String name`, `Album* albums`, etc.).

So the only required textual change to make **Instance 1** conform to **Grammar 2** is: remove the `mongobean` keyword before each bean name, keeping everything else (including whitespace/formatting/comments) unchanged.

import java.util.*

package org.musicdb {
	/**
	 * A musical performer
	 */
	Artist {
		String name
		Album* albums
		
		Iterable<Track> getOevre() {
			albums.map[tracks].flatten
		}
	}

	 * A record, CD, DVD or another collection of tracks
	 */
	Album {
		String title
		int year 
		Track* tracks
		
		Duration getDuration() {
			tracks.map[duration].reduce[Duration a, Duration b| a + b]
		}
	}
	
	Track {
		String title
		Duration duration
	}
	
	Duration {
		int seconds
	
		void fromString(String duration) {
			val split = duration.split(':')
			seconds = Integer::parseInt(split.get(0)) * 60 + Integer::parseInt(split.get(1))
		}		
		
		String toString() {
			Integer::toString(seconds / 60) + ':' + Integer::toString(seconds % 60)
		}
		
		Duration operator_plus(Duration other) {
			new Duration() [
				seconds = this.seconds + other.seconds
			]
		}
	}
}
