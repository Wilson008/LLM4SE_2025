Differences between Grammar 1 and Grammar 2:

1. **MongoBean header changed**
   - **Grammar 1:** `MongoBean: 'mongobean' name=ValidID '{' ... '}'`
   - **Grammar 2:** `MongoBean: name=ValidID '{' ... '}'`
   - Effect: the literal keyword **`'mongobean'` is removed** from bean declarations.

2. **MongoProperty type extended (OR alternative added)**
   - **Grammar 1:** `type=JvmTypeReference (many?='*')? name=ValidID;`
   - **Grammar 2:** `(type=JvmTypeReference | inlineType=MongoBean) (many?='*')? name=ValidID;`
   - Effect: properties may now use either a normal JVM type reference or an inline embedded `MongoBean`.  
   - Per your rule (2), since this is a new optional/OR capability, we **do not instantiate** the new `inlineType=MongoBean` alternative; we keep the existing JVM types as-is.

3. **Everything else stays the same**
   - Imports, packages, operations, wildcard qualified names remain unchanged.

### Instance 2 (modified to follow Grammar 2)

// Instance 2
import java.util.*

package org.musicdb {
	/**
	 * A musical performer
	 */
	Artist {
		String name
		Album* albums
		
		Iterable<Track> getOevre() {
			albums.map[tracks].flatten
		}
	}  
	 
	/**
	 * A record, CD, DVD or another collection of tracks
	 */
	Album {
		String title
		int year 
		Track* tracks
		
		Duration getDuration() {
			tracks.map[duration].reduce[Duration a, Duration b| a + b]
		}
	}
	
	Track {
		String title
		Duration duration
	}
	
	Duration {
		int seconds
	
		void fromString(String duration) {
			val split = duration.split(':')
			seconds = Integer::parseInt(split.get(0)) * 60 + Integer::parseInt(split.get(1))
		}		
		
		String toString() {
			Integer::toString(seconds / 60) + ':' + Integer::toString(seconds % 60)
		}
		
		Duration operator_plus(Duration other) {
			new Duration() [
				seconds = this.seconds + other.seconds
			]
		}
	}
}
