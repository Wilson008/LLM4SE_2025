## Key differences: Grammar 2 vs Grammar 1

### 1) `ChkVariableDeclaration` syntax changed (arrays + special value block)
**Grammar 1**
```antlr
type=ChkTypeReference (arr+='[' (sz+=CHK_NUMBER)? ']')* name=ValidID
...
('value' '{' spValue=SpValue '}')?;
```
- Array brackets allowed **with optional size**: `[]` or `[n]`.
- The special-value block keyword is **`value`**.

**Grammar 2**
```antlr
type=ChkTypeReference ('[' sz+=CHK_NUMBER ']')* name=ValidID
...
('sp' '{' spValue=SpValue '}')?;
```
- Array brackets now require a size each time: must be `[something]`, no empty `[]`.
- The special-value block keyword changed from **`value` → `sp`**.

Impact: some Grammar 1 declarations like `int[] a` would no longer parse; and `value { ... }` must be written as `sp { ... }`.

---

### 2) `Helper.select` changed: now includes a range after `:`
**Grammar 1**
```antlr
select='select' '(' vars+=ValidID (',' vars+=ValidID)* ')'
```

**Grammar 2**
```antlr
select='select' '(' vars+=ValidID (',' vars+=ValidID)* ':' ((var3=ChkVariables) '..' (var4=ChkVariables)) ')'
```
- `select(...)` now **requires** `: <something> .. <something>` inside the parentheses.
- This is a breaking change for instances like your Instance 1, e.g.:
  - `select(k,l)` (Grammar 1) would need to become something like `select(k,l: 1..n)` (exact intended usage depends on your semantics).

---

### 3) `FormatExpression` redesigned; introduced `FormatVariable`
**Grammar 1**
```antlr
FormatExpression:
  num = CHK_NUMBER '(' (var+=ValidID ('[' sz+=CHK_NUMBER ']')* ('<' (count+=CHK_NUMBER) '>')?
    (',' var+=ValidID ('[' sz+=CHK_NUMBER ']')* ('<' (count+=CHK_NUMBER) '>')?)*)? ')';
```
- Format variables were plain `ValidID` plus optional `[size]...` and optional `<count>`.

**Grammar 2**
```antlr
FormatExpression:
  num = CHK_NUMBER '(' (var+=FormatVariable (',' var+=FormatVariable)*)? ')';

FormatVariable:
  fv=ChkVariable ('<' (count=CHK_NUMBER) '>')?;
```
- A format variable is now a full `ChkVariable` (so it can include prefixes like `in.` / `out.` / `ans.`, backtick, indexing, etc.), plus optional `<count>`.
- Removed the old “`ValidID` then optional `[...]` sizes” structure from format directly.

Impact: existing things like `y<n>` from Instance 1 still fit (as `ChkVariable` + `<count>`), but Grammar 2 is more uniform and expressive here.

---

### 4) Everything else is effectively unchanged
Rules such as `MR`, `FollowUp`, `Property`, `Check`, expression/loop structure, operators, `Limit`, `SpValue`, and numeric/token rules appear the same, aside from the changes listed above.
